# AgentOS Studio + Database Setup

## Tính năng

Sau khi tích hợp Database, AgentOS hiện có:

| Tính năng | Mô tả |
|-----------|------|
| **Studio** | Visual builder để tạo agents, teams, workflows |
| **Registry** | Quản lý available tools, models, databases |
| **Session Tracking** | Lưu tất cả agent runs + traces |
| **Version Control** | Save, publish, restore versions |

---

## 1. Setup Database (PostgreSQL)

### Tùy chọn A: Local PostgreSQL (Windows)

```powershell
# Cài PostgreSQL từ: https://www.postgresql.org/download/windows/
# Chọn default port 5432

# Sau khi install, tạo database + user:
psql -U postgres

# Trong psql shell:
CREATE USER agno WITH PASSWORD 'agno_password';
CREATE DATABASE agentosagno OWNER agno;
GRANT ALL PRIVILEGES ON DATABASE agentosagno TO agno;
\q
```

### Tùy chọn B: Docker (Nhanh nhất)

```bash
docker run -d \
  --name agentosagno-db \
  -e POSTGRES_USER=agno \
  -e POSTGRES_PASSWORD=agno_password \
  -e POSTGRES_DB=agentosagno \
  -p 5432:5432 \
  postgres:16-alpine

# Verify:
psql -h localhost -U agno -d agentosagno -c "SELECT 1"
```

### Tùy chọn C: Docker Compose (Recommended)

Thêm vào `docker-compose.yml`:

```yaml
services:
  agentosagno-db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: agno
      POSTGRES_PASSWORD: agno_password
      POSTGRES_DB: agentosagno
    ports:
      - "5432:5432"
    volumes:
      - agentosagno_db_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U agno"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  agentosagno_db_data:
```

## 4. Kết nối Control Plane

### Step 1: Truy cập Control Plane

Vào https://os.agno.com → Đăng nhập

### Step 2: Add New OS

Click "Add new OS" → Điền:

| Field | Value |
|-------|-------|
| **Name** | HITC AgentOS |
| **Endpoint** | http://localhost:8000 (dev) hoặc https://your-domain.com (prod) |
| **Environment** | Local (dev) hoặc Live (prod) |

Click "CONNECT" → AgentOS sẽ:
- Xuất hiện trong dashboard
- Registry được load vào Control Plane
- Studio sẵn sàng để build

## 5. Studio: Build Agents Visually

### Workflow

1. **Open Studio** (trong Control Plane)
2. **Build** → Drag & drop agents, teams, workflows
3. **Configure** → Setup instructions, tools, models
4. **Save Draft** → Test before publish
5. **Test** → Interact via Chat Page
6. **Publish** → Move to production
7. **API Access** → `GET /agents/{agent_id}/published` để access published version

### Ví dụ: Create Data Query Agent in Studio

1. Drag **Agent** block từ palette
2. Set name: "Data Query Agent"
3. Set instructions: "Truy vấn dữ liệu nhân viên từ HITC system"
4. Add tools: Kéo MCPTools từ Registry
5. Set model: OpenAI-compatible (Qwen)
6. Connect tools → click "Test"
7. Save draft → Publish

## 7. Database Schema

AgentOS tự động tạo tables:

```sql
-- Agents
CREATE TABLE agents (
  id UUID PRIMARY KEY,
  name VARCHAR,
  description TEXT,
  instructions TEXT,
  tools JSONB,
  model JSONB,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

-- Teams (multi-agent compositions)
CREATE TABLE teams (
  id UUID PRIMARY KEY,
  name VARCHAR,
  agents JSONB,
  created_at TIMESTAMP
);

-- Workflows (step-based execution)
CREATE TABLE workflows (
  id UUID PRIMARY KEY,
  name VARCHAR,
  steps JSONB,
  created_at TIMESTAMP
);

-- Runs (execution history)
CREATE TABLE runs (
  id UUID PRIMARY KEY,
  agent_id UUID,
  input TEXT,
  output TEXT,
  trace JSONB,
  session_id VARCHAR,
  created_at TIMESTAMP
);

-- Versions (for agents, teams, workflows)
CREATE TABLE versions (
  id UUID PRIMARY KEY,
  entity_id UUID,
  entity_type VARCHAR,
  version_number INT,
  data JSONB,
  created_at TIMESTAMP
);
```

## 8. Troubleshooting

### Database Connection Error

```
⚠ Cannot connect to AgentOS DB (postgresql+psycopg://...),
using SQLite fallback
```

**Fix:**
```powershell
# Kiểm tra PostgreSQL đang chạy
psql -h localhost -U agno -d agentosagno -c "SELECT 1"

# Nếu error, restart PostgreSQL:
# Windows: Services → PostgreSQL → Restart
# Docker: docker restart agentosagno-db

# Update .env với đúng credentials
```

### Studio Registry Empty

**Problem**: Studio không thấy tools/models

**Fix**:
```python
# Kiểm tra Registry được pass vào AgentOS
# workflow/agents.py → create_agent_os_app()

registry = Registry(
    name="...",
    tools=[MCPTools(url=settings.MCP_GATEWAY_URL)],  # ← Check MCP_GATEWAY_URL
    models=[OpenAILike(...)],  # ← Check LLM settings
    dbs=[db],
)
```

### Control Plane Cannot Connect

**Problem**: "Cannot connect to http://localhost:8000"

**Fix**:
1. Verify AgentOS running: `curl http://localhost:8000/health`
2. For remote access, use ngrok: `ngrok http 8000`
3. Update Control Plane endpoint to ngrok URL

## 9. Production Deployment

### Docker Deployment

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# Install dependencies
COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

# Ensure DB is ready
RUN python -c "from app.core.config import settings; print(settings.AGENTOSAGNO_DB_URL)"

CMD ["python", "run.py"]
```

### Environment Variables (Production)

```env
# Production DB (managed PostgreSQL like AWS RDS)
AGENTOSAGNO_DB_HOST=agentosagno-prod.abc123.us-east-1.rds.amazonaws.com
AGENTOSAGNO_DB_PORT=5432
AGENTOSAGNO_DB_USER=agno_prod
AGENTOSAGNO_DB_PASSWORD=<random_secure_password>
AGENTOSAGNO_DB_NAME=agentosagno_prod

# Control Plane
AGENTOSAGNO_ENDPOINT=https://hitc-agentosagno.yourdomain.com
AGENTOSAGNO_API_KEY=<optional_api_key>
```

### Kubernetes (Optional)

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: agentosagno
spec:
  replicas: 2
  selector:
    matchLabels:
      app: agentosagno
  template:
    metadata:
      labels:
        app: agentosagno
    spec:
      containers:
      - name: agentosagno
        image: your-registry/agentosagno:latest
        ports:
        - containerPort: 8000
        env:
        - name: AGENTOSAGNO_DB_HOST
          valueFrom:
            secretKeyRef:
              name: agentosagno
              key: db-host
        - name: AGENTOSAGNO_DB_PASSWORD
          valueFrom:
            secretKeyRef:
              name: agentosagno
              key: db-password
```

## 10. Quick Start (All-in-one)

```powershell
# 1. Start PostgreSQL (Docker)

  -e POSTGRES_USER=agno `
  -e POSTGRES_PASSWORD=agno_password `
  -e POSTGRES_DB=agentosagno `
  -p 5432:5432 `

# 2. Create .env
"AGENTOSAGNO_DB_HOST=localhost
AGENTOSAGNO_DB_PORT=5432
AGENTOSAGNO_DB_USER=agno
AGENTOSAGNO_DB_PASSWORD=agno_password
AGENTOSAGNO_DB_NAME=agentosagno" | Out-File .env

# 3. Install/Update requirements
pip install -r requirements.txt

# 4. Run AgentOS
python run.py

# 5. Verify (new terminal)
curl http://localhost:8000/health
curl http://localhost:8000/agents/

# 6. Visit https://os.agno.com
#    → Add new OS → http://localhost:8000
#    → Open Studio → Start building!
```
